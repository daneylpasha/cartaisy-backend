# Shopify Tenant Client Audit

## Scope and method

This audit searched the backend for Shopify Admin API and Storefront API usage, singleton/global clients, and direct `process.env` Shopify credentials. It is intentionally documentation-only and does not refactor checkout, auth refresh logic, dashboard, backend architecture, or other flows.

Commands used during the audit:

- `rg -n "getShopifyClient|shopifyService|storefrontService|SHOPIFY_|process\.env\.SHOPIFY|new shopify|shopifyApi|Storefront|storefront|adminApi|accessToken" src --glob '!public/swagger.json'`
- `rg -n "shopify|Shopify|SHOPIFY|storefront|Storefront|admin\.rest|Graphql|graphql|accessToken|storefrontAccessToken" src scripts sync-products-now.js test-*.js docs public/swagger.json`

## Executive summary

The main tenant-isolation risk is that several mobile-facing Storefront flows still call the singleton `shopifyStorefrontService` instance, which initializes Storefront and Admin clients from process-wide environment variables at module construction time. A store-specific Storefront helper exists, but only the collection-products endpoint is currently wired to use it when `x-store-id` is present.

The Admin API side has both a store-specific client helper and a legacy compatibility helper. The legacy helper chooses the first connected Shopify store from MongoDB and is used by many sync/inventory/order helper functions, making those functions unsafe for shared multi-tenant usage unless callers are explicitly single-tenant/admin-only or are updated to pass a store ID.

## Recommended next implementation ticket

Create a small implementation ticket to make mobile Storefront reads tenant-scoped first, before touching checkout/payment completion:

1. Define how mobile requests provide store context consistently, preferably via an authenticated tenant/store claim or required `x-store-id` header.
2. Add store-specific Storefront methods for search, product detail, favorites, home-screen, recommendations, and cart reads/writes.
3. Remove production fallback to `SHOPIFY_STOREFRONT_ACCESS_TOKEN` when a Store lacks `shopify.storefrontAccessToken`.
4. Add tests that fail when mobile Storefront routes call singleton/env credentials without store context.

A separate follow-up should address checkout/order completion and legacy Admin sync helpers because those flows carry higher regression risk and were explicitly out of scope for this issue.

## Risks

- This is a static audit. Some call paths may be admin-only or unreachable depending on routing and deployment configuration.
- The audit intentionally does not change runtime behavior, so existing tenant-isolation risks remain until follow-up implementation work is completed.
- Several files contain checkout, dashboard, or auth-adjacent code that should be changed only in focused follow-up tickets with targeted tests.
